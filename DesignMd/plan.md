一、整体规划与基础搭建（已经完成，设计时不需要再考虑）  
1. 明确技术栈与部署方式（微信小程序原生 + Spring Boot 后端）  
2. 规划系统模块：用户模块、家庭（组）模块、家庭重要事项模块、提醒与推送模块、公共支撑模块（配置、日志等）  
3. 搭建基础项目骨架：  
   - 后端 Spring Boot 工程、数据库设计与连接配置  
   - 小程序项目初始化（app 配置、全局样式、基础路由结构）  

二、用户体系与微信一键登录  
1. 接入微信小程序登录流程（code 换取 session / openid）  
2. 设计并实现用户表结构（含微信标识、基本信息、状态等）  
3. 完成后端登录接口与登录态管理（token / session 方案确定与实现）  
4. 小程序前端登录按钮与一键登录流程打通（含失败与重试处理）  

三、家庭（组）管理：创建与加入  
1. 设计家庭（组）表结构（家庭名称、户主、成员数量、创建时间等）  
2. 实现创建家庭接口与前端页面（首登用户可创建“xxx家庭”）  
3. 设计加入家庭流程与接口（通过家庭编号/搜索等方式申请加入）  
4. 设计并实现户主审批机制（查看待审批成员、通过/拒绝）  
5. 成员与家庭关系表设计，实现成员只能属于/查看自己家庭数据的权限控制  

四、成员权限与家庭主页  
1. 定义成员角色与权限（户主、普通成员，后续可扩展）  
2. 后端统一权限校验（所有家庭相关接口基于当前用户所属家庭过滤数据）  
3. 设计家庭主页展示：家庭基本信息、成员列表、重要事项概览等  

五、家庭重要事项基础功能  
1. 设计重要事项数据结构（事项名称、类别、创建人、家庭 ID、状态等核心字段）  
2. 后端实现重要事项的创建、查询、删除等基础接口  
3. 前端实现创建重要事项页面与列表展示页面  
4. 限制删除权限：只有创建人可以删除，其他成员仅可查看  

六、重要事项的提醒配置功能  
1. 在重要事项中增加提醒配置字段：  
   - 是否周期提醒、提醒周期（按日/周/月/年）  
   - 提醒时间类型（农历/阳历）  
   - 提前几天提醒、提醒次数  
   - 提醒范围（提醒所有人 / 指定某个成员）  
2. 前端提醒配置 UI 设计与交互实现（创建/编辑事项时配置提醒）  
3. 后端保存与解析提醒配置逻辑（含数据校验与默认值策略）  

七、提醒任务调度与到期判断  
1. 设计提醒任务的调度方案（定时任务 / 任务队列等）  
2. 实现基于日/周/月/年周期的到期计算逻辑（同时支持农历与阳历）  
3. 实现“提前几天提醒”“提醒几次”的判断与状态记录  
4. 只针对有效的、未关闭的提醒生成待发送任务  

八、微信小程序消息推送实现  
1. 选型并确认使用的微信订阅消息 / 模板消息类型与字段  
2. 后端实现消息推送服务：根据到期提醒向目标用户推送小程序消息  
3. 完成与微信平台的配置与鉴权（appId、secret、消息模板配置等）  
4. 小程序前端引导用户开启订阅消息授权  

九、提醒结果处理与再次提醒  
1. 创建人收到提醒后，可在小程序中选择“关闭本次提醒/不再提醒”等操作  
2. 关闭本次提醒后，到下一个周期自动重新生成提醒任务（若整体提醒未关闭）  
3. 若用户选择彻底关闭提醒，则更新事项提醒配置，后续不再生成任务  

十、日志、监控与运维支持（首版简化）  
1. 基本操作日志记录（登录、创建家庭、创建/删除事项等关键行为）  
2. 任务调度与消息推送的错误日志与简单监控  
3. 提供简单的后台查询接口（用于问题排查和后期扩展后台管理）  

十一、测试与上线  
1. 编写核心流程的接口与前端功能测试用例（登录、建家、加家、建事项、提醒）
